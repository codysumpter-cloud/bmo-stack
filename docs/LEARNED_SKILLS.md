# Learned skills

This repo now has a bounded learning loop.

## Rule

Reusable procedures may be extracted from verified work into `context/learned/generated-skills/`, but they are **advisory** until reviewed.

## Promotion criteria

A generated skill should only be treated as active when all of these are true:

- the source task was verified
- the pattern is narrow and reusable
- the pattern has appeared more than once
- a human reviewed the generated skill
- the generated skill does not replace a clearer source-of-truth document

## Non-goals

- no silent self-modification
- no automatic registration into runtime routing
- no completion claims based on generated memory alone
